# work-log: 업무일지 & 성과 보고서 & 성과 정리 스킬

## Goal

5개 업무 도구(GitHub, Slack, Jira, Google Calendar, Notion)에서 활동을 자동 수집하여 세 가지 모드의 보고서를 생성하는 Claude Code 스킬.

- **Journal 모드**: 시간순 업무일지 + 인사이트 분석
- **Impact 모드**: 역량 영역별 성과 분류 + 임팩트 측정 보고서
- **Organize 모드** (v0.3.0): 유연한 성과 카테고라이징
  - **report**: 사내 보고용 — 경영진 시각의 6개 비즈니스 가치 카테고리
  - **resume**: 이력서용 — 13개 관점의 엔지니어 역량 프레임워크 + 스코어 + AI 리라이팅

## 성과 프레임워크

### Impact 모드 — 역량 영역 (4개)
- 기술 / 코드 / 설계
- 제품
- 커뮤니케이션
- 문화

### Impact 모드 — 임팩트 태그 (5개)
- 💰 매출: 매출/전환/재구매 변화
- 💰 비용: 외주/운영비 절감
- ⏱ 시간: 기계 & 인력 투입 시간 절감
- 🛡 리스크: 장애/보안 리스크 감소
- ✅ 품질: 결함/재작업 감소

### Organize report — 비즈니스 가치 카테고리 (6개)
- 매출/딜리버리
- 운영 효율화
- 장애 & 보안 리스크 감소
- 비용 절감
- 채용 & 홍보
- 팀 빌딩/리더십

### Organize resume — 엔지니어 역량 프레임워크 (13개 관점)
**기술/코드/설계 (7)**: 라이브러리 내부 구현 이해, 모범사례 비교 & 설계, 리소스 최적화, 배포 임팩트 측정, 장애 감지 & 대응, RCA & 재발 방지, 성능 최적화/운영 자동화
**제품 (2)**: 비즈니스 성장 기여, 담당 제품 규모 & 영향력
**커뮤니케이션 (2)**: 데이터 기반 의견 조율, 기록을 통한 문맥 전달
**문화 (2)**: 가고 싶은 팀 만들기, 정확한 피드백

## 핵심 원칙
- 역량 영역으로 빠짐없이 분류하고, 임팩트 태그로 강도를 증명
- 임팩트를 자동 감지하되, 못 찾으면 유저에게 물어봄
- 측정 불가 항목에는 "다음에 이렇게 측정하세요" 제안 포함
- 이력서 항목은 1-10 스코어로 강도 평가, 약한 항목은 AI 리라이팅 제안

## 데이터 소스 & 수집 범위

| 소스 | 수집 항목 | on/off |
|------|----------|--------|
| GitHub | 커밋, PR(생성/리뷰/머지), 코드리뷰 | 설정 가능 |
| Slack | 메시지 요약 | 설정 가능 |
| Jira | 할당 티켓 상태 | 설정 가능 |
| Google Calendar | 참석/주최 미팅 | 설정 가능 |
| Notion | 수정된 페이지 | 설정 가능 |

## 실행

- 수동: `/work-log`, `/work-log impact`, `/work-log impact Q1`
- 수동: `/work-log organize report`, `/work-log organize resume Q1`
- 자동: 매주 토요일 scheduled task (journal 또는 organize 모드)
